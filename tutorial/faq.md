# Proma 常见问题与解决方案

> 本文档持续更新中，收录用户最常遇到的问题和解决步骤。

## 一、网络与连接问题

### 1. 请求超时 / 连接失败
**症状**：对话发送后长时间无响应，或出现"连接超时"错误
**原因**：通常是网络代理未配置或 AI 服务暂时不可用
**解决步骤**：
1. 检查网络是否正常（能否打开 claude.ai 或对应 AI 服务页面）
2. 进入 设置 → 代理 → 确认代理地址和端口配置正确
3. 如已配置代理，尝试关闭后重新打开，或切换代理节点
4. 检查 AI 渠道密钥是否仍然有效（设置 → 渠道 → 测试连接）

### 2. API Key 无效 / 401 错误
**症状**：发送消息后提示 "Invalid API Key" 或 401 错误
**解决步骤**：
1. 进入 设置 → 渠道 → 点击对应渠道的编辑按钮
2. 重新填写 API Key（注意不要有多余空格）
3. 点击"测试连接"确认 Key 可用
4. 如果是 Anthropic 官方 Key，确认账户余额充足

### 3. 429 / Rate Limit 错误
**症状**：提示"Too many requests"或 429 错误
**原因**：API 请求频率超过渠道限制
**解决步骤**：
1. 等待 1-2 分钟后重试
2. 如频繁出现，考虑切换到其他渠道或降低使用频率
3. 付费套餐用户可联系渠道提高限额

## 二、对话与生成问题

### 4. 对话中途中断 / 生成停止
**症状**：AI 回复突然停止，内容不完整
**解决方案**：
- 直接在输入框发送"继续"或"continue"，AI 会接续生成
- 或点击消息旁的"重新生成"按钮

### 5. 上下文丢失 / AI 记不住之前的内容
**症状**：AI 好像忘记了对话前面的内容
**原因**：上下文长度设置限制了能看到的历史消息数量
**解决步骤**：
1. 点击对话顶部的上下文设置图标
2. 将上下文长度调为"无限"或增大轮数
3. 如对话很长，考虑新建对话并在开头提供关键背景

### 6. 回复速度慢
**可能原因**：模型较大（如 Opus）、网络延迟、上下文过长
**建议**：
- 速度优先的任务切换到 Sonnet 或 Haiku 模型
- 使用上下文分隔线，减少每次携带的历史长度

## 三、Skills 与工具问题

### 7. Skill 没有触发
**症状**：发送了应该触发某个 Skill 的消息，但 AI 没有调用它
**解决步骤**：
1. 确认 Skill 已安装（设置 → 工具 → Skills 列表）
2. 检查 Skill 的触发描述，确认消息内容符合触发条件
3. 尝试更明确的表述，或直接在消息中用 `/skill名称` 显式触发
4. Agent 模式下确认 Skill 已在当前工作区启用

### 8. MCP 工具连接失败
**症状**：Agent 模式下工具调用失败，提示 MCP 连接错误
**解决步骤**：
1. 进入 设置 → 工具 → MCP，查看各工具连接状态
2. 检查 MCP 服务是否正在运行（部分 MCP 需要本地后台服务）
3. 重启 Proma 后重新连接
4. 查看工具调用日志了解具体错误信息

## 四、Agent 运行问题

### 9. Agent 任务执行报错并停止
**解决步骤**：
1. 查看错误信息，区分工具调用失败还是权限问题
2. 权限问题：在对话中回复"允许"或调整权限设置
3. 网络问题：等待片刻后发送"继续"重试

### 10. 定时任务没有按时运行
**排查步骤**：
1. 确认 Proma 在任务触发时处于运行状态（定时任务需要 App 保持后台运行）
2. 检查任务状态（侧边栏 → 自动任务 → 查看对应任务状态和最近记录）
3. 确认任务所用模型渠道余额充足且可用

### 11. Agent 对话没有进展 / 卡住了
**解决方案**：
- 发送"继续"让 Agent 继续执行
- 如 Agent 陷入死循环，点击"停止"后重新描述任务
- 把复杂任务拆分为更小的子任务逐步完成

## 五、安装与启动问题

### 12. macOS 提示"无法打开，因为无法验证开发者"
**解决方法**：
右键点击 Proma.app → 选择"打开" → 在弹出窗口中点击"打开"即可正常使用

### 13. Windows 安装后无法启动
**排查步骤**：
1. 以管理员身份重新运行安装程序
2. 检查 Windows Defender 是否误报，添加到信任列表
3. 确认系统满足最低配置要求

## 六、数据与设置问题

### 14. 如何迁移 / 备份数据
- 进入 设置 → 数据迁移，可导出完整备份
- 备份文件包含所有对话历史、渠道配置和 Skills

### 15. 更新后配置丢失
**原因**：极少数情况下更新可能重置部分设置
**处理**：
1. 进入 设置 → 数据迁移 → 导入备份文件
2. 如无备份，重新在 设置 → 渠道 中配置 API Key
